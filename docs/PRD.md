# PRD：線上抽籤系統

## 1. 專案概述

### 背景與動機
目前市面上缺乏一個最單純且快速的開箱即用抽籤工具。我們希望提供一個只要貼上名單就能馬上開始操作的抽籤系統，免去繁雜的設定與無關的功能，讓任何需要隨機抽選結果的人都能輕鬆使用。

### 目標用戶
- **所有人**

### 核心價值主張
- **簡單直覺**：不需學習成本，快速完成設定。
- **趣味動態**：透過抽籤動畫增加過程的刺激感與樂趣。

---

## 2. 功能需求

### 主要功能清單
1. **輸入活動資訊**：設定抽籤活動的主題或基本說明。
2. **簡單貼上參加者名單**：讓使用者直接透過文字框貼上多筆參加者資料。
3. **設定抽出數量**：自行決定本次要隨機抽出的中獎者人數。
4. **即時執行的抽籤動態結果**：點擊抽籤後，直接在畫面上呈現具有動態效果的抽籤過程與結果。
5. **檢視與儲存抽籤結果**：在抽籤結束後展示結果列表，並自動儲存以供後續查閱。

### 使用者故事 (User Stories)
- 作為 **[所有人]**，我希望 **[能輸入活動資訊]**，以便 **[明確標示這個抽籤活動的目的]**。
- 作為 **[所有人]**，我希望 **[能簡單貼上參加者名單]**，以便 **[省去一筆筆建檔的麻煩，快速載入名單]**。
- 作為 **[所有人]**，我希望 **[能自由設定抽出數量]**，以便 **[配合不同情境下需要的當選人數]**。
- 作為 **[所有人]**，我希望 **[體驗即時執行的抽籤動態結果]**，以便 **[讓抽籤過程更有趣，並確保視覺上的公平與懸念]**。
- 作為 **[所有人]**，我希望 **[能檢視並儲存抽籤結果]**，以便 **[在抽籤結束後留存中獎紀錄備查]**。

---

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Flask。
- **前端渲染**：使用 Jinja2 與原生 Web 技術 (HTML/CSS/JS)。
- **資料庫**：使用 SQLite 儲存。

### 效能與安全考量
- 確保名單中沒有空白或是無效字元。
- 確保輸入的「抽出數量」大於 0 且不大於「參加者總人數」。
- 動態效果應流暢，不會因為參加人數過高而導致前端效能崩潰。

---

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須擁有)
- 輸入活動資訊的表單欄位。
- 支援多行文字貼上的參加者名單輸入框。
- 設定抽出數量的數字輸入框。
- 「開始抽籤」的按鈕，且點擊後會呈現畫面動態變化並產出最終中籤名單。

### Should Have (應該擁有)
- 將抽出的結果保存至資料庫中（搭配 SQLite）。
- 若名單或抽出數量不合法則跳出系統錯誤提示字眼。

### Nice to Have (未來可以擁有)
- 能夠回顧以前所有建立並抽籤過的歷史紀錄。
- 把結果匯出為檔案。
